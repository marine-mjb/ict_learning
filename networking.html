🌐 IP Addresses & Subnetting

Every device on a network needs a unique IP address. IPv4 (like 192.168.1.1) uses 32 bits, IPv6 (like 2001:db8::1) uses 128 bits. Subnet masks separate network and host portions, enabling efficient routing.

# IPv4 Address Classes Class A: 1.0.0.0 - 126.255.255.255 (large networks) Class B: 128.0.0.0 - 191.255.255.255 (medium networks) Class C: 192.0.0.0 - 223.255.255.255 (small networks) # Private IP ranges (not routable on internet) 10.0.0.0/8 → 10.0.0.0 - 10.255.255.255 172.16.0.0/12 → 172.16.0.0 - 172.31.255.255 192.168.0.0/16 → 192.168.0.0 - 192.168.255.255
💡 Pro tip: Use `ipconfig` (Windows) or `ifconfig` (Linux/Mac) to see your device's IP address. 127.0.0.1 is "localhost" — your own computer!

🔍 DNS: The Phonebook of Internet

DNS (Domain Name System) translates human-friendly domain names (google.com) into IP addresses (142.250.190.46). Without DNS, we'd have to memorize number strings for every website!

# DNS lookup process 1. Browser checks cache → 2. OS checks hosts file 3. DNS resolver query → 4. Root nameserver 5. TLD nameserver (.com) → 6. Authoritative nameserver 7. IP returned → 8. Browser connects # Common public DNS servers Google: 8.8.8.8, 8.8.4.4 Cloudflare: 1.1.1.1, 1.0.0.1 OpenDNS: 208.67.222.222, 208.67.220.220
✅ Best practice: Change your DNS to Cloudflare (1.1.1.1) or Google (8.8.8.8) for faster, more private browsing!
🏗️ OSI Model & Protocols

The OSI model has 7 layers: Physical, Data Link, Network, Transport, Session, Presentation, Application. Key protocols: HTTP/HTTPS (web), TCP/UDP (transport), IP (network), Ethernet (data link).

# OSI Layers (memorize: "Please Do Not Throw Sausage Pizza Away") 7 - Application (HTTP, FTP, DNS) 6 - Presentation (SSL/TLS, encryption) 5 - Session (NetBIOS, RPC) 4 - Transport (TCP, UDP) 3 - Network (IP, ICMP, routing) 2 - Data Link (Ethernet, MAC addresses) 1 - Physical (cables, hubs, signals) # TCP vs UDP TCP: Reliable, ordered, error-checked (web, email, file transfer) UDP: Fast, no guarantee (streaming, gaming, VoIP)
🚀 Remember: Routers operate at Layer 3 (Network), Switches at Layer 2 (Data Link). Understanding layers helps troubleshoot network issues!
🔌 Network Devices & Topologies

Routers connect different networks. Switches connect devices within same network. Hubs broadcast to all ports (outdated). Access Points provide Wi-Fi. Common topologies: Star, Bus, Ring, Mesh.

# Device functions 🖧 Router → Forwards packets between networks (home WiFi router) 🖧 Switch → Connects devices, learns MAC addresses 🖧 Hub → Broadcasts to all ports (security risk, low efficiency) 🖧 Modem → Converts ISP signal to Ethernet 🖧 AP → Wireless access point for Wi-Fi # ping command tests connectivity ping google.com # tests reachability tracert google.com # shows route (Windows) traceroute google.com # same on Linux/Mac
⚡ Network tip: Use `ping 8.8.8.8` to test internet connectivity. If it works but `ping google.com` fails, your DNS is the problem!
